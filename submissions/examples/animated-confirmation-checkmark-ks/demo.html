Animated Confirmation Checkmark

A two-phase micro-interaction for form submissions, payments, and success states. The ring completes a full circle before the checkmark draws in.

Variants

Success
Error
Warning
Loading

Sizes

Small
Medium
Large

Integration

HTML

<!-- Success confirmation -->
<div class="confirm-container" aria-live="polite"
     aria-label="Payment successful">
  <svg class="confirm-ring confirm-ring-success"
       viewBox="0 0 80 80" aria-hidden="true">
    <circle cx="40" cy="40" r="35"
            class="confirm-ring-track"/>
    <circle cx="40" cy="40" r="35"
            class="confirm-ring-fill"/>
  </svg>
  <svg class="confirm-check" viewBox="0 0 80 80"
       aria-hidden="true">
    <polyline points="22,42 35,55 58,26"
              class="confirm-check-path"/>
  </svg>
</div>

CSS Animation

/* Two-phase: ring draws first, then checkmark appears */
.confirm-ring-fill {
  stroke-dasharray: 220;
  stroke-dashoffset: 220;
  animation: draw-ring 0.6s ease forwards;
}

.confirm-check-path {
  stroke-dasharray: 60;
  stroke-dashoffset: 60;
  animation: draw-check 0.4s ease 0.5s forwards;
}

@keyframes draw-ring {
  to { stroke-dashoffset: 0; }
}

@keyframes draw-check {
  to { stroke-dashoffset: 0; }
}

Accessibility

/* Respect user motion preferences */
@media (prefers-reduced-motion: reduce) {
  .confirm-ring-fill,
  .confirm-check-path {
    animation: none;
    stroke-dashoffset: 0;
    opacity: 1;
  }

  .confirm-check-path {
    animation: confirm-check-appear 0.01ms forwards;
  }
}

Lifecycle States

Processing →
Success

This is the standard two-step flow: show a loading spinner during processing, then transition to the success checkmark when the operation completes. Use JavaScript to swap between these markup states.
