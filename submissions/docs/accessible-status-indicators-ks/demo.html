How It Works

The Accessibility Animation Spectrum

Accessible animation is not a binary on/off switch. It is a spectrum that adapts to the user's environment:

Full Animation Pulsing dots, color shifts, smooth transitions for spatial awareness
Reduced Motion Opacity-only fade pulse, no translation, slower timing
No Animation Static colors, borders, or text-only indicator. Zero frame activity.
Forced Colors System colors (CanvasText, LinkText), border-based indicators, zero reliance on color alone

CSS Pattern

/* 1. Base: animated */
.status-dot {
  width: 10px;
  height: 10px;
  border-radius: 50%;
  background: #10b981;
  animation: pulse-glow 2s ease-in-out infinite;
}

/* 2. Reduced motion: swap to opacity pulse, no transform */
@media (prefers-reduced-motion: reduce) {
  .status-dot {
    animation-name: pulse-opacity;
  }
}

/* 3. Forced colors: use system colors + border */
@media (forced-colors: active) {
  .status-dot {
    background: LinkText;
    border: 2px solid CanvasText;
    animation: none;
  }
}

ARIA Pattern

<!-- Use role="status" for live region that does not interrupt -->
<span class="status-dot" role="status"
      aria-label="User is online"></span>

<!-- Use aria-live="polite" for update announcements -->
<div aria-live="polite" aria-atomic="true">
  <span class="sr-only">3 new notifications</span>
</div>

Best Practices

  • Never rely on color alone. Combine color with shape (dot vs ring vs diamond), text labels, and ARIA attributes.
  • Prefer prefers-reduced-motion: reduce to swap spatial transforms for opacity fades — users still get motion feedback, just without vestibular triggers.
  • Add forced-colors: active support. Strip out background-color and box-shadow; use border, outline, and system color keywords (CanvasText, LinkText, Highlight).
  • Use semantic live regions. role="status" for persistent indicators, aria-live="polite" for transient updates. Avoid aria-live="assertive" — it interrupts screen-reader users.
  • Keep animation short. Loop durations under 2 seconds for pulse effects. Longer animations (5s+) feel sluggish and make the user wait.
  • WCAG 2.3.3 compliance: If a user interaction triggers animation, provide a way to disable it. For purely decorative status animations, no toggle is required.
