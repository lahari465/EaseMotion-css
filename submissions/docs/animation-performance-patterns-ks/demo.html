CSS Property Performance Reference

Composited Only ✅

No layout, no paint. 60fps on any device.

  • transform (translate, rotate, scale)
  • opacity
  • filter (blur, drop-shadow, brightness)
  • backdrop-filter

Paint Only ⚠️

GPU accelerates paint. Often 60fps, but can jank.

  • color
  • background-color
  • box-shadow
  • border-color
  • outline-color

Layout + Paint ❌

Triggers full pipeline. Avoid in animations.

  • width / height
  • top / right / bottom / left
  • margin / padding
  • border-width
  • font-size

Applying to EaseMotion Components

Class Issue Better Approach
.ease-typewriter-loop Animates width + will-change: width Use clip-path: inset() for text reveal, zero layout cost
.ease-reveal will-change applied permanently Move to :hover / .active state; the component already resets on .ease-reveal-active
.ease-password-strength-bar Transitions width Use transform: scaleX() with transform-origin: left
.ease-hover-glow Uses filter: drop-shadow() Already optimal — drop-shadow is composited. Good pattern.
.ease-btn Transitions background-color + box-shadow Acceptable for buttons. Paint is fast on interactive elements.

Performance Checklist

  • Prefer transform and opacity. These are the only two properties that skip layout and paint entirely. Animate position with translateX/Y, size with scale.
  • Scope will-change to interaction states. Never apply it permanently to a base class. Use :hover or :focus-visible. Remove it when the animation ends.
  • Use transform: scaleX() for progress bars. Never animate width — it triggers layout on every frame. scaleX() with transform-origin: left is visually identical and composited.
  • Use filter: drop-shadow() for hover glows. Unlike box-shadow, it's composited and not clipped by overflow: hidden parents.
  • Animate clip-path for reveals, not width/height. Typewriter effects, mask reveals, and expand transitions should use clip-path.
  • Test on low-end hardware. Chrome DevTools → Performance → CPU throttling (6x slowdown). If it drops below 30fps, switch to composited-only properties.
  • Respect prefers-reduced-motion. Even composited animations can cause discomfort. Always provide a reduced-motion alternative.
